# code_reviews

This directory stores markdown reports generated by the `crazyShell Reviewer` agent.

## What it does

- reviews the HyperTerm Canvas repository in proposition-only mode
- inspects architecture, correctness, testing gaps, documentation drift, and maintenance risks
- writes one markdown report per review run
- records the date and reference commit in both the filename and the report frontmatter

## Naming

Report filenames use this shape:

```text
YYYY-MM-DDTHH-mm-ss.sssZ__<short-commit>.md
```

Example:

```text
2026-04-18T08-10-00.000Z__ec7bc7aa8a10.md
```

## Commands

Run one review:

```bash
bun run review:agent
```

Run the polling watcher (default: every 300 seconds):

```bash
bun run review:agent:watch
```

Custom polling interval:

```bash
bun run review:agent:watch --poll-seconds=900
# or
CRAZYSHELL_REVIEW_POLL_SECONDS=900 bun run review:agent:watch
```

## Notes

- The watcher triggers when the current HEAD commit has no review report yet.
- Reports are proposition-only; the reviewer should not modify code.
- Any detected mutation outside `code_reviews/` is treated as a hard failure and the review run exits non-zero.
